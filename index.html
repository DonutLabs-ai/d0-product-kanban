🔑 Issue 拆解原则

Milestone → Epic → Scope → Issue 的拆解方法 · 每个 Issue 必须能回答矩阵坐标的推进
Level 1
Milestone
矩阵上的目标格子
= 一个季度的大目标
例: P1×S1 (30d)
Level 2
Epic
按四层架构拆分
Command / Trust / Cognition / Culture
例: Trust → GRPS 集成
Level 3
Scope
功能模块 / 技术范围
1 Scope ≈ 1-2 sprint
例: GRPS → D0 Bot 集成
Level 4
Issue
工程师可接的粒度
必须回答: 把我们从哪推到哪
例: GRPS 软约束方法论
Milestone → Epic· Epic → Scope· Scope → Issue
🔑 Issue 验收标准
每个 Issue 必须能回答：「这把我们从矩阵的 P?×S? 推到 P?×S?」
回答不了 → 说明拆解还不够清晰，或者这个 issue 不在战略路径上。
